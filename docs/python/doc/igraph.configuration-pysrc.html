 70      General ideas 
 71      ============= 
 72   
 73      The configuration of igraph is stored in the form of name-value pairs. 
 74      This object provides an interface to the configuration data using the 
 75      syntax known from dict: 
 76   
 77        >>> c=Configuration() 
 78        >>> c["general.verbose"] = True 
 79        >>> print c["general.verbose"] 
 80        True 
 81   
 82      Configuration keys are organized into sections, and the name to be used 
 83      for a given key is always in the form C{section.keyname}, like 
 84      C{general.verbose} in the example above. In that case, C{general} is the 
 85      name of the configuration section, and C{verbose} is the name of the key. 
 86      If the name of the section is omitted, it defaults to C{general}, so 
 87      C{general.verbose} can be referred to as C{verbose}: 
 88   
 89        >>> c=Configuration() 
 90        >>> c["verbose"] = True 
 91        >>> print c["general.verbose"] 
 92        True 
 93   
 94      User-level configuration is stored in C{~/.igraphrc} per default on Linux 
 95      and Mac OS X systems, or in C{C:\\Documents and Settings\\username\\.igraphrc} 
 96      on Windows systems. However, this configuration is read only when C{igraph} 
 97      is launched through its shell interface defined in L{igraph.app.shell}. 
 98      This behaviour might change before version 1.0. 
 99   
100      Known configuration keys 
101      ======================== 
102   
103      The known configuration keys are presented below, sorted by section. When 
104      referring to them in program code, don't forget to add the section name, 
105      expect in the case of section C{general}. 
106   
107      General settings 
108      ---------------- 
109   
110      These settings are all stored in section C{general}. 
111   
112          - B{shells}: the list of preferred Python shells to be used with the 
113            command-line C{igraph} script. The shells in the list are tried one 
114            by one until any of them is found on the system. C{igraph} functions 
115            are then imported into the main namespace of the shell and the shell 
116            is launched. Known shells and their respective class names to be 
117            used can be found in L{igraph.app.shell}. Example: 
118            C{IPythonShell, ClassicPythonShell}. This is the default, by the way. 
119          - B{verbose}: whether L{igraph} should talk more than really necessary. 
120            For instance, if set to C{True}, some functions display progress bars. 
121   
122      Application settings 
123      -------------------- 
124   
125      These settings specify the external applications that are possibly 
126      used by C{igraph}. They are all stored in section C{apps}. 
127   
128          - B{image_viewer}: image viewer application. If set to an empty string, 
129            it will be determined automatically from the platform C{igraph} runs 
130            on. On Mac OS X, it defaults to the Preview application. On Linux, 
131            it chooses a viewer from several well-known Linux viewers like 
132            C{gthumb}, C{kuickview} and so on (see the source code for the full 
133            list). On Windows, it defaults to the system's built-in image viewer. 
134   
135      Plotting settings 
136      ----------------- 
137   
138      These settings specify the default values used by plotting functions. 
139      They are all stored in section C{plotting}. 
140   
141          - B{layout}: default graph layout algorithm to be used. 
142   
143          - B{mark_groups}: whether to mark the clusters by polygons when 
144            plotting a clustering object. 
145   
146          - B{palette}: default palette to be used for converting integer 
147            numbers to colors. See L{colors.Palette} for more information. 
148            Valid palette names are stored in C{colors.palettes}. 
149   
150          - B{wrap_labels}: whether to try to wrap the labels of the

152            Default: C{False}.
